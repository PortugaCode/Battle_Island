Github 팀작업 규칙

# 깃허브 규칙
1. 각자 씬에서만 작업
2. 작업 후 E1I3 브랜치에 자기 브랜치 commit → push → pull request 보내기 (merge는 팀장이)
3. Asset에 만든 파일(프리팹, 스크립트, 오디오 등) 뒤에 _이름 쓰기
Ex) PlayerMove_oh (이 스크립트는 오중근 말고는 노터치 터치해야할 시 연락요망)
4. Asset 폴더에 이미 있는 데이터는 최대한 수정하지 말고 수정 필요하면 말하기
5. 문제 발생 시 연락하기

# 작업과정
1. main에서 다 같이 pull
2. 각자 main에서 branch 생성 (★ 자기 branch에서 작업, main에서 작업 절대금지)
3. 유니티 에디터에서 각자 씬 생성 (이름 안겹치게)
4. 각자 씬에서만 작업
5. 작업한 내용 알아서 백업하고 규칙(2) 따라서 commit
6. 작업한 내용 merge 끝나면 branch 삭제되고 다시 1부터 반복

# Commit할 때 방법
Summary / Description (Description에만 한글 가능)
[Feat] Playermove   (플레이어 움직임 완성)
[Feat] PlayerShoot
[Fix] PlayerShoot

feat   새로운 기능에 대한 커밋
fix   버그 수정에 대한 커밋
build   빌드 관련 파일 수정 / 모듈 설치 또는 삭제에 대한 커밋
chore   그 외 자잘한 수정에 대한 커밋
ci   ci 관련 설정 수정에 대한 커밋
docs   문서 수정에 대한 커밋
style   코드 스타일 혹은 포맷 등에 관한 커밋
refactor   코드 리팩토링에 대한 커밋
test   테스트 코드 수정에 대한 커밋
perf   성능 개선에 대한 커밋
